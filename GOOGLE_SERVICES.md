# Google Services Integration

EQUA heavily leverages the Google Cloud and AI ecosystem to deliver a production-grade AI bias firewall. The use of Google services is central to the application's ability to provide real-time, explainable, and scalable fairness audits.

## 1. Google Gemini API (Core Feature)
The heart of EQUA's explainability engine is powered by the **Google Gemini API** (specifically the `gemini-2.0-flash-lite` model). 

### How it is used:
When the Counterfactual Simulator intercepts a biased decision, the frontend calculates the feature disparity delta and sends a highly structured prompt directly to the Gemini API via the `@google/generative-ai` SDK.
Gemini acts as an **AI Fairness Auditor**, generating a real-time, human-readable compliance narrative that explains *why* the underlying model rejected an applicant based purely on the swapped protected attribute. It also generates actionable ML remediation suggestions (e.g., removing proxy variables).

**Why Gemini?**
- **Speed:** The Flash models provide the sub-second latency required for a real-time firewall.
- **Reasoning:** Gemini excels at translating raw SHAP feature attributions into readable legal/compliance narratives required by the EU AI Act.

## 2. Firebase Hosting (Deployment)
To ensure high availability and a seamless global experience for the Solution Challenge judges, the EQUA React SPA (Single Page Application) is designed to be deployed via **Firebase Hosting**. 

### How it is used:
Firebase Hosting serves our optimized Vite production bundle over a global CDN. This ensures the dashboard loads instantly and the complex SVG animations render smoothly without server-side bottlenecks. It also provides automatic SSL provisioning, ensuring that all API calls to the Gemini models remain secure.

## 3. Vertex AI (Simulated Ecosystem)
While EQUA acts as the firewall, it is designed to sit in front of models deployed on **Vertex AI**.
Our "Retraining Loop" dashboard visualizes how EQUA would integrate with Vertex AI pipelines:
- Capturing data drift.
- Automatically triggering a Vertex AI pipeline to retrain the model.
- Evaluating the new model via Vertex Explainable AI before deploying the updated endpoint.

## 4. Google Cloud KMS (Simulated ecosystem)
The Fairness Certificates generated by EQUA are conceptually signed and hashed using **Google Cloud Key Management Service (KMS)** to ensure non-repudiation of compliance audits.
